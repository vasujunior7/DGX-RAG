How It Works

1. User Input

User pastes PDF URL and asks question in web UI

2. Document Processing

Backend fetches PDF, applies legal-aware chunking, builds FAISS index

3. Smart Retrieval

Query encodes → kNN search over FAISS → top 5 relevant chunks

4. LLM Generation

Anthropic/Gemini models generate grounded answers with source attribution

5. Response Delivery

Answer returns with metadata and renders in chat interface

Key ML Components: SentenceTransformers (all-MiniLM-L6-v2) • FAISS vector search • Legal chunker • ThreadPoolExecutor batch processing

Risks & Mitigation

Large PDF Latency

Risk: 500+ page documents cause timeout

Mitigation: Background indexing + pre-warming + progress indicators

LLM API Dependency

Risk: Anthropic/Gemini outages block responses

Mitigation: Multi-provider fallback + local model option + circuit breakers

Memory Constraints

Risk: FAISS index size grows beyond RAM

Mitigation: Disk-based storage + index sharding + compression

Data Privacy (PII)

Risk: Sensitive information in policy documents

Mitigation: PII redaction + encryption at rest + access controls

Network Reliability

Risk: PDF fetch failures from external URLs

Mitigation: Retry logic + timeout handling + local caching

Our Approach: Honest assessment + realistic mitigation plans + continuous monitoring
